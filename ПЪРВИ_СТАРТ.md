# Първи старт – за да работи регистрацията

Съобщението *„Не може да се свърже със сървъра“* означава, че **бекендът не работи**. Регистрацията и входът изискват бекенд сървър да е пуснат.

## Стъпка 1: MySQL

1. Пусни **MySQL** (XAMPP, WAMP, или инсталиран MySQL сървър).
2. Отвори MySQL (например phpMyAdmin или командния ред) и създай базата:
   ```sql
   CREATE DATABASE IF NOT EXISTS carpool_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. Провери, че имаш потребител с парола за MySQL. В `backend/src/main/resources/application.yml` са зададени:
   - хост: `localhost`, порт: `3306`
   - база: `carpool_db`
   - потребител: `root`, парола: `root`  
   Ако при теб са различни, промени ги в `application.yml`.

## Стъпка 2: Бекенд (Spring Boot)

Отвори **нов терминал** в папката на проекта и пусни бекенда:

**Вариант А – с Maven (ако имаш `mvn` в системата):**
```bash
cd backend
mvn spring-boot:run
```

**Вариант Б – с файла `backend\start.bat` (Windows):**  
Двойно кликване на `backend\start.bat` (изисква инсталиран Maven в системата).

**Вариант В – от IntelliJ IDEA / Eclipse:**  
Отвори проекта `backend` като Maven проект и стартирай класа `CarpoolBackendApplication` (Run).

Изчакай да се появи съобщение от типа:
```text
Started CarpoolBackendApplication in ... seconds
```
Това означава, че сървърът работи на **http://localhost:8080**.

**Проверка:** Отвори в браузър **http://localhost:8080/api/v1/health** – трябва да видиш `{"status":"UP"}`. Ако страницата не се зарежда, бекендът не работи или е на друг порт.

## Стъпка 3: Фронтенд

В **друг терминал**:

```bash
cd frontend
npm install
npm run dev
```

Отвори в браузър **http://localhost:5173** и опитай отново **Регистрация** или **Вход**.

---

**Обобщение:** Регистрацията ще работи само когато бекендът е пуснат (порт 8080) и MySQL е стартиран с базата `carpool_db`.

---

## Бекендът ми е на друг порт/адрес

Фронтендът по подразбиране извиква **http://localhost:8080** (стандартният порт на Spring Boot). Ако твоят бекенд работи на друг порт (напр. 8081) или на друг компютър:

1. В папката **frontend** създай файл **`.env`** (копирай от `.env.example`).
2. Задай правилния адрес, напр.:
   ```
   VITE_API_URL=http://localhost:8081
   ```
   или
   ```
   VITE_API_URL=http://192.168.1.5:8080
   ```
3. **Спри** фронтенда (Ctrl+C) и пусни отново: `npm run dev`.

В грешката вече се показва към кой адрес се опитва връзката – ако е различен от твоя бекенд, промени `.env` и рестартирай.
